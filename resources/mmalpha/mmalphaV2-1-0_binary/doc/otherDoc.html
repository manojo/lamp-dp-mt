The Alpha Langage

  • How to use MMAlpha to model dataflow systems. This was published in ASAP05. In this 2.1.0 version, the synPeriodic command allows one to generate Vhdl for dataflow systems. See the documentation in the doc/PeriodicSystems directory. This is quite difficult to use currently, and quite fragile.
  • How to build a new distribution, using the CopyMMAlpha.m file. The CopyMMAlpha.m file was significantly modified for version 2.1. This is a more a memo for myself than something for a user.
  • The MacOS X implementation. This document reports some notes that I took while adapting MMAlpha to Darwin. Unsure of the content.
  • How to install MMAlpha. This documentation is up to date, except maybe for Windows versions, which I was not able to test recently.
  • A note on the Polylib. Explains how to install a new version of PolyLib. This is needed before compiling a new version of Domlib.
  • A description of the Meta generator
    • Explains how to use the Meta package to create quickly translators from Alpha to some specialized languages. This is how the Vhdl generator is done.
  • A note on the Scheduler for the users. Concerns only the \texttt{schedule} function. Another scheduler, named \texttt{scd} is also available. Both do basically the same thing, but some options differ.
  • The Synthesis program. This presents the syn command.
  • Synthesis of periodic systems. This presents the synPeriodic command. Fragile...
  • Should be, but is not up-to-date... A description of the AlpHard language, a subset of Alpha to model hardware.
  • The Pipeline command. Notes taken recently, but file was not checked. Pipeline is a very problematic transformation...
  • How to install the Domlib.
